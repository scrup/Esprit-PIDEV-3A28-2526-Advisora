Symfony Guide - Gestion Ressource
=================================

Objectif
--------
Mettre en place le module web Gestion Ressource dans Symfony en suivant la meme structure que Gestion Projet, sans modifier la logique metier des autres modules.
Le perimetre couvre maintenant :

- CRUD Ressource
- CRUD Fournisseur

Perimetre retenu
----------------
- CRUD Ressource prioritaire
- CRUD Fournisseur prioritaire
- meme structure visuelle et technique que Gestion Projet
- controle de saisie conserve
- integration minimale dans le menu front/back
- aucune modification de logique metier dans les autres gestions

Structure reprise de Gestion Projet
-----------------------------------
Le module Ressource/Fournisseur a ete construit en reprenant le meme schema que le module Projet :

- un controller principal avec front + back + manage + form
- des templates Twig separes pour front et back
- le meme fichier CSS principal : public/css/project-management.css
- le meme script de validation formulaire : public/js/project-form-validation.js
- le meme principe de filtres, cartes, pages de detail et page de gestion
- un dashboard back de module pour gerant/admin avec acces separe a la fenetre Ressources et a la fenetre Fournisseurs

CRUD Ressource ajoute
---------------------
Routes principales :

- /back/resources/dashboard
- /resources
- /back/resources
- /resources/{id}
- /resources/new
- /resources/{id}/manage
- /back/resources/{id}/manage
- /resources/{id}/edit
- /resources/{id}/delete

Comportement :

- dashboard back module Ressource/Fournisseur
- lecture catalogue front
- lecture back office
- detail ressource
- ajout ressource
- modification ressource
- suppression avec protection si la ressource est liee a des projets ou au mini-shop
- visualisation explicite de la liaison ressource <-> projet via project_resources

CRUD Fournisseur ajoute
-----------------------
Routes principales :

- /back/suppliers
- /suppliers/new
- /back/suppliers/{id}/manage
- /suppliers/{id}/edit
- /suppliers/{id}/delete

Comportement :

- lecture back office des fournisseurs
- ajout fournisseur
- modification fournisseur
- suppression fournisseur si aucune ressource n est rattachee
- passerelle directe fournisseur -> ajout ressource avec fournisseur preselectionne

Controle de saisie ajoute
-------------------------
- nom de la ressource obligatoire
- longueur max du nom : 160
- statut obligatoire
- fournisseur obligatoire
- quantite >= 0
- prix > 0
- URL image optionnelle mais valide si renseignee
- suppression securisee par jeton CSRF

Controle de saisie Fournisseur
------------------------------
- nom fournisseur obligatoire
- nom fournisseur unique
- quantite >= 0
- entreprise fournisseur obligatoire
- email obligatoire et valide
- localisation obligatoire
- numero de telephone obligatoire et valide
- suppression securisee par jeton CSRF

Reservation ressource cote client
---------------------------------
La reservation client a ete branchee dans le module Ressource Symfony pour suivre la logique de l ancien projet Java.

- fiche ressource front : /resources/{id}
- action de reservation : /resources/{id}/reserve
- liste client des reservations : /resources/reservations
- edition client d une reservation : /resources/reservations/{projectId}/{resourceId}/edit
- role autorise : client uniquement
- quantite obligatoire et controlee par rapport au stock disponible reel
- projet selectionnable
- si aucun projet n est choisi, le systeme reutilise le dernier projet client ou cree automatiquement "Reservation Ressources"
- liaison en base : table project_resources
- si une quantite > 1 est reservee, la colonne qtyAllocated peut etre ajoutee automatiquement a project_resources

Historique back des reservations
--------------------------------
- liste globale : /back/resources/reservations
- edition manager/admin : /back/resources/reservations/{projectId}/{resourceId}/edit
- suppression manager/admin : /back/resources/reservations/{projectId}/{resourceId}/delete
- le gerant et l admin voient toutes les reservations
- le client ne voit que ses propres reservations
